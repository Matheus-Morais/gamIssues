• O que é o GamIssue?

    Octalysis, Elementos de Jogos, Issues, GitLab e Devs!

    Em poucas palavras o GamIssue é a gamificação da Issues de uma equipe de desenvolvimento.

    Mas, para tornar isso realidade foi necessario três contextos, sendo eles: A fabrica de software do CEULP/ULBRA, Gitlab e o Octalysis.

    A idéia principal desta gamificação é trazer elementos de jogos para melhorar a produtvidade de uma equipe de desenvolvimento.
    Para isso foi usado elementos de jogos como: Experience Points, Níveis, Missões e Ranking.

  • Qual a mecanica do GamIssue?

    Adiquira missões, finalize elas, se torne o maior dev de todos os tempos!

    No GamIssue você tem pontos de experiência(XP), níveis e missões.
    Para conseguir XP e subir de nivel, você precisa finalizar as missões adquiridas, mas, para ficar melhor de entender vamos mostrar para você como se tornar o maior dev de todos os tempos em 5 etapas.

    • Primeira Etapa
      Realize seu cadastro e faça login!
      Primeiro faça seu cadastro, vá na parte superior da tela e clique em "Login", logo você vai ver onde se cadastrar, se ainda não viu, fica embaixo do botão "Entrar".
      Para se cadastrar é preciso o Nome, Sobrenome, Email, Usuario, Senha e o Private Token do GitLab.
      Importante
      Você deve utilizar o Usuario que você usa para logar no do GitLab!
      Você deve gerar um token de acesso no GitLab. Para gerar esse token faça login no GitLab e vá nas configurações de sua conta, depois em "Tokens de acesso" no menu lateral da pagina do Gitlab. Essa é pagina de criação do token, basta seguir esses passos:
      - Passo 1: Dê um nome (de sua preferência) para esse token!
      - Passo 2: Escolha uma data de vencimento (31/12/2018), se não tiver data o token vai valer para sempre.
      - Passo 3: Em "Scopes" clique em "api".
      - Passo 4: Por fim em "Create personal access token"!
      Agora o token já deve ter aparecido e você vai poder terminar seu cadastro!
      Mas antes de se cadastrar, atualize sua foto do GitLab, pois ela será usada para indicar sua colocação no Ranking e no seu Perfil.
      Agora com seus dados cadastrados, basta fazer login e seguir para a proxima etapa!
    • Segunda Etapa
      Pegue uma Missão!
      Veja como é simples. As missões serão todas as issues que você esta como Assignee ou responsável.
      Basta seguir os seguintes passos:
      - Passo 1: Clique em "Missões", no canto superior direito da tela.
      - Passo 2: Escolha um Projeto e clique em "missões" para ver as missões disponíveis/abertas/finalizadas para aquele projeto.
      - Passo 3: Você pode clicar em "Pegar Todas as Missões" para pegar todas as missões daquele projeto de uma vez, ou, clicar em "Pegar Missão" que ai pegaria apenas 1 por vez.
      Apenas as issues com status Open vão aparecer.
      Dica: Pegue todas as Missões antes de trabalhar nelas!
    • Terceira Etapa
      Desenvolva!
      Nesta etapa é necessario "apenas" isso. Desenvolva e feche as missões no GitLab.
      Importante
      Serão somadas apenas as missões que você esta como Assignee, não importa quem feche a missão.
    • Quarta Etapa
      Finalize uma Missão!
      Você pegou as missões, agora é a hora de finalizar! Para isso siga os seguintes passos:
      - Passo 1: Escolha um Projeto e clique em "missões" para ver as missões disponíveis/abertas/finalizadas para aquele projeto.
      - Passo 2: Você pode clicar em "Finalizar Todas as Missões" para finalizar todas as missões daquele projeto de uma vez, ou, clicar em "Finalizar Missão" dentro de "Missões em Aberto" que ai finalizaria apenas 1 por vez.
      OBS: Cada missão vale 80 XP, porem, se finalizada depois da data prevista, esta missao vai vale apenas metade disso, ou seja, 40 XP.
    • Quinta Etapa
      Verifique seu lugar no ranking!
      Assim que as missões são finalizadas os pontos para cada uma são contados, e entra em ação o sistema de Ranking.
      Rankings de:
      - Experiência
      - Missões Fechadas no Tempo Certo
      - Missões Fechadas
      - Missões Abertas!
